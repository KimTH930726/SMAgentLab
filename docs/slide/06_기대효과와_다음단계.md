> 슬라이드 수: 1장
> 목표: 이 시스템이 가져오는 변화와 가까운 다음 단계를 간결하게 전달한다

---

## [슬라이드 6-1] 기대효과와 다음 단계

### 제목
**이 시스템이 가져오는 변화, 그리고 다음**

### 본문 내용

**지금 달라지는 것**

- 질문 → 시스템 즉시 답변
- 흩어진 지식 → 검색 가능한 조직 자산
- 몰랐던 공백 → 가시화 + 채움
- 반복 문의·숙련자 부재 → **온보딩 비용 절감**

**데이터가 쌓이면 열리는 것**
- 운영 매뉴얼 자동화 — 반복 Q&A 기반으로 초안 생성 제안 → 사람이 검토·등록
- MCP 확장 — 장애·배포 이력, 모니터링 알림 등 운영 시스템과 결합한 답변으로 확장

**가까운 다음 단계**

| 카테고리 | 항목 | 기대효과 |
|----------|------|---------|
| **지식 현행화** | Confluence Webhook | 문서 수정 시 자동 재수집 → 수동 재등록 부담 제거 |
| **통합 알림 채널 확장** | 관제·VOC 메일링 데이터 수집 연동 | 흩어진 알림을 하나의 채널로 모아 이슈를 선제적으로 감지 |

### 발표자 노트
> "결국 이 시스템이 만드는 변화는 단순합니다 — 질문하면 바로 답이 나오고, 지식은 사람이 아니라 조직에 쌓이고, 몰랐던 공백은 눈에 보여서 채울 수 있게 됩니다. 그 결과가 온보딩 비용 절감으로 이어집니다.
> 가까운 다음 단계는 지식이 outdated 되지 않게 자동으로 현행화하는 것, 그리고 지금은 메일링 등으로 흩어진 관제·VOC 데이터를 통합 알림 채널로 모아 확장하는 것입니다. 둘 다 이미 갖춰진 인프라 위에서 설정 수준으로 빠르게 실현할 수 있습니다."
