## [슬라이드 6-1] 기대효과와 다음 단계

### 제목
**이 시스템이 가져오는 변화, 그리고 다음**

### 본문 내용

**지금 달라지는 것**

```
[Before]               [After]
  질문 → 사람 찾기       질문 → 시스템 즉시 답변
  문서 → 검색 어려움     문서 → 등록 후 자동 검색
  지식 → 개인 의존       지식 → 조직 자산화
  공백 → 모름           공백 → 가시화 + 채움
```
- 반복 운영 문의에 즉각적인 1차 답변, 숙련자 부재 시에도 지식 접근 가능 — 온보딩 비용 절감

**데이터가 쌓이면 열리는 것**
- 통합 알림 채널 — 메일링 등으로 흩어진 관제·VOC 데이터를 하나의 채널로 모아 분석 → 이슈를 선제적으로 알림
- 운영 매뉴얼 자동화 — 반복 Q&A 기반으로 초안 생성 제안 → 사람이 검토·등록
- MCP 확장 — 장애·배포 이력, 모니터링 알림 등 운영 시스템과 결합한 답변으로 확장

**가까운 다음 단계**

| 카테고리 | 항목 | 기대효과 |
|----------|------|---------|
| **지식 현행화** | Confluence Webhook | 문서 수정 시 자동 재수집 → 수동 재등록 부담 제거 |
| **공백 가시화** | 지식 공백 클러스터링 | `no_knowledge` 질문을 주제별로 묶어 무엇을 먼저 채워야 할지 명확화 |
| **검색 품질** | 한국어 리랭커 탑재 | Top-K 재정렬로 더 관련성 높은 청크 전달 → 답변 정확도 향상 |

### 발표자 노트
> "당장은 반복 문의 처리와 지식 체계화가 달라지고, 데이터가 쌓이면 관제·VOC 통합 알림 분석이나 운영 매뉴얼 자동화처럼 지금까지 사람이 하던 일을 AI가 초안 수준에서 지원하게 됩니다.
> 근거리 항목들은 이미 인프라가 갖춰져 있어 설정 수준으로 빠르게 실현 가능합니다.
> 결국 이 프로젝트에서 배운 건 하나입니다 — AI 자체보다 '우리 데이터를 AI에게 먹이는 구조'가 먼저라는 것, 그 구조만 갖추면 확장은 생각보다 빠릅니다."
