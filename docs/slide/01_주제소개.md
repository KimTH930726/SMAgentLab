## [슬라이드 1-2] 오늘 발표 범위

### 키 메시지
RAG란 무엇인지, 왜 '지속 가능'이 중요한지

### 본문 내용

**RAG(Retrieval-Augmented Generation)란?**
- LLM(대형 언어 모델)은 모르는 내용도 그럴듯하게 지어내는 환각(Hallucination) 문제가 있음
- 이를 막기 위해, 답변 생성 시 사전에 등록된 내부 지식을 검색해 근거로 활용하는 방식 = RAG
- 환각 방지는 물론, 사내 도메인 지식까지 답변에 반영 가능

**왜 '지속 가능'이 중요한가?**
- 한 번 구축하고 끝나는 시스템 → 시간이 지나면 지식이 outdated 되어 품질 저하
- 지식이 축적될수록, 피드백이 쌓일수록 좋아지는 선순환 구조가 필요

**오늘 다룰 내용**
1. 왜 만들었는가 (배경)
2. 어떻게 만들었는가 (구조 + 기술)
3. 구축 과정에서 배운 것
4. 앞으로 어디까지 갈 수 있는가

### 발표자 노트
> "RAG라는 단어가 생소하신 분도 계실 수 있는데, 쉽게 말하면 'AI가 우리 회사 내부 문서를 보면서 답변하는 것'입니다.
> 오늘 발표의 핵심은 이 시스템이 한 번 만들고 끝이 아니라, 계속 사용하면서 더 좋아지도록 설계했다는 점입니다."
