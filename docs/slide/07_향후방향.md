# 슬라이드 7 — 향후 방향

> 슬라이드 수: 1~2장
> 목표: 슬라이드 5에서 도출한 3가지 교훈을 로드맵으로 연결 — "배운 것을 어떻게 이어갈 것인가"
> 방침: 단기/실현 가능한 것 중심, 너무 먼 미래는 최소화

---

## [슬라이드 7-1] 향후 고도화 방향 — 3가지 교훈의 연장선

### 제목
**배운 것을 다음 단계로**

### 본문 내용

> 슬라이드 5에서 정리한 세 가지 교훈을 각각의 방향으로 이어간다

**① 지식 파이프라인 — 더 자동화, 더 낮은 진입 장벽**

| 우선순위 | 항목 | 내용 |
|----------|------|------|
| 근거리 | Confluence Webhook | 페이지 수정 시 자동 재수집 → 지식 현행화 자동화 |
| 근거리 | 지식 갭 자동 클러스터링 | `no_knowledge` 질문들을 AI가 주제별로 묶어 "이런 내용이 부족합니다" 제안 |
| 중기 | 청킹 전략 고도화 | Parent-Child 청킹 — 긴 문서의 세부 내용도 정확히 검색되도록 |
| 중기 | 능동 학습 루프 | 불확실한 답변에서 피드백 요청 → few-shot 자동 후보 생성 |

**② 맥락 보강 — LLM이 우리 시스템을 더 잘 이해하게**

| 우선순위 | 항목 | 내용 |
|----------|------|------|
| 근거리 | 한국어 리랭커 탑재 | 한국어 특화 CrossEncoder(`mmarco-mMiniLMv2`) → 검색 결과 재정렬 품질 향상 |
| 중기 | 평가 지표 체계화 | 골든셋 Q&A 100~200개 구축 → Recall@K, MRR 정량 측정 |
| 중기 | 자동 품질 저하 감지 | 주간 배치로 Recall 측정 → 기준치 이하 시 알림 |

**③ 거버넌스 — AI 판단의 투명성과 통제를 더 정밀하게**

| 우선순위 | 항목 | 내용 |
|----------|------|------|
| 근거리 | 파이프라인 감사 로그 고도화 | 어떤 컨텍스트가 LLM에 전달됐는지 이력 저장 → 사후 검증 가능 |
| 중기 | 비용 모니터링 대시보드 | 네임스페이스별 토큰 사용량 추적 → 비용 최적화 의사결정 지원 |
| 장기 | MCP 도구 실행 감사 | HTTP 호출 결과·파라미터 이력 → AI가 외부에 뭘 요청했는지 추적 |

**+α — 허브가 성숙하면, MCP로 붙일 수 있는 것들이 늘어난다**

```
지식·용어집·질의 로그가 쌓인 허브 위에
  MCP 도구를 연결하면 범위가 급격히 확장된다

  예) 장애 이력 조회 API    → "지난달 이 오류 몇 번 났어?" 자연어로
      배포 이력 시스템       → "최근 배포와 장애 시점 겹치는지 확인해줘"
      모니터링 알림 채널     → 실시간 상태를 LLM 컨텍스트에 직접 주입
```

- 파이프라인이 먼저 갖춰져야 MCP 확장이 의미 있다 — 지식 없는 도구 연결은 절반짜리
- 운영 시스템 전반을 자연어로 접근하는 통합 운영 어시스턴트로 발전 가능

### 발표자 노트
> "방향을 단기/중기/장기 시간축 대신 3가지 교훈의 축으로 나눴습니다.
> 이렇게 보면 각 기능이 왜 필요한지가 더 명확해집니다.
> 근거리 항목들은 이미 인프라가 갖춰져 있어서 설정이나 소규모 작업으로 빠르게 실현 가능합니다.
> 그리고 허브 데이터가 쌓일수록 MCP로 연결할 수 있는 운영 시스템 활용 범위도 같이 넓어집니다."

## [슬라이드 7-2] 마무리

### 제목
**AI 도입, 어떻게 시작할 것인가**

### 본문 내용

```
지식 파이프라인 구축
  → 데이터가 AI에게 전달 가능한 형태로 축적
  → LLM 맥락 보강이 가능해진다
  → MCP, Agentic 확장이 수월해진다
        │
        └── 단, 자율화될수록
            AI 판단에 대한 거버넌스·투명성이 반드시 함께
```

> **"AI 자체보다 '우리 데이터를 AI에게 먹이는 구조'가 먼저다.
> 그리고 그 과정에서 사람이 최소한의 개입으로 통제할 수 있어야 지속된다."**

### 발표자 노트
> "마지막으로 남기고 싶은 메시지는 이겁니다.
> 어떤 AI 모델을 쓰느냐보다, 우리 데이터를 얼마나 잘 구조화해서 AI에게 전달할 수 있느냐가 실질적인 품질을 결정합니다.
> 그 파이프라인이 잘 갖춰지면 이후 확장은 생각보다 빠릅니다.
> 다만, AI가 자율적으로 판단하는 부분이 많아질수록 백오피스에서 통제하고 감사할 수 있는 거버넌스 구조를 함께 가져가야 한다는 것,
> 이 시스템을 만들면서 가장 크게 배운 점입니다. 감사합니다."
