## [슬라이드 3-1] 시스템 전체 구성

### 제목
**Ops-Navigator 전체 아키텍처**

### 다이어그램 (텍스트 표현 → 시각화 도구로 변환)

```
사용자 (웹 브라우저)
        │  질문 입력 / 파일 등록
        ▼
  ┌─────────────────────┐
  │  Frontend           │  React + nginx  (port 8501)
  │  - 채팅 UI           │
  │  - 지식 관리 어드민   │
  └──────────┬──────────┘
             │ REST API / SSE 스트리밍
             ▼
  ┌─────────────────────┐
  │  Backend            │  FastAPI  (port 8000)
  │  - KnowledgeRAG     │
  │  - 인증/권한         │
  └──────────┬──────────┘
             │
     ┌───────┴────────┐
     ▼                ▼
PostgreSQL          사내 LLM
+ pgvector          (DevX Gateway)
(지식/임베딩 저장)   devx-gw.shinsegae-inc.com
```

### 핵심 포인트
- 모든 컴포넌트가 **Docker Compose**로 단일 서버에 배포
- 폐쇄망 환경: 이미지를 tar.gz로 반입, `docker load`로 설치
- 임베딩 모델(`paraphrase-multilingual-mpnet-base-v2`, 768차원)은 컨테이너 내 탑재

### 발표자 노트
> "전체 시스템은 프론트엔드, 백엔드, DB 세 개의 컨테이너로 구성되어 있습니다.
> 외부 인터넷 없이도 동작하며, 사내 LLM 게이트웨이와만 통신합니다."

## [슬라이드 3-2] KnowledgeRAG 에이전트

### 제목
** KnowledgeRAG 에이전트**

### 본문 내용

| 구분 | KnowledgeRAG 에이전트 |
|------|----------------------|
| 용도 | 운영 지식/절차/FAQ 질문 |
| 동작 | 등록된 지식 검색 → LLM 답변 |

### 발표자 노트
> "오늘 발표에서 소개해드릴 내용은 KnowledgeRAG입니다.

## [슬라이드 3-3] 데이터 흐름 — 질문부터 답변까지

### 제목
**질문 하나가 답변이 되기까지**

### 흐름 다이어그램

```
사용자 질문
    │
    ▼
① 임베딩 변환 (768차원 벡터)
    │
    ▼
② 용어 매핑 (Glossary) — 사내 용어 자동 인식
    │  "쿠폰" → 매핑 용어 추가
    ▼
③ 하이브리드 검색
    ├─ 벡터 검색 (의미 유사도) × 0.7
    └─ 키워드 검색 (정확한 단어 매칭) × 0.3
    │
    ▼
④ 관련 지식 Top-K 선별 + 신뢰도 판단
    │
    ▼
⑤ Few-shot 예제 병합 (유사 Q&A 사례)
    │
    ▼
⑥ LLM 답변 생성 (SSE 스트리밍)
    │
    ▼
⑦ 로그 기록 + 피드백 수집
```

### 발표자 노트
> "단순히 '비슷한 문서 찾아서 LLM에 넣는 것' 이상의 단계들이 있습니다.
> 특히 용어 매핑과 하이브리드 검색이 사내 도메인 정확도를 높이는 핵심입니다."

## [슬라이드 3-4] 시연

### 제목
**실제 동작 화면**

### 구성
- **[영상 삽입]** 녹화된 시연 영상 재생 (2~3분)

### 시연 시나리오 (영상 녹화 시 참고)
1. 사용자가 운영 이슈 관련 질문 입력
   - 예: "결제 오류 발생 시 1차 확인 항목은 무엇인가요?"
2. 실시간 스트리밍으로 답변 생성되는 모습
3. 답변 하단의 근거 문서 확인
4. 피드백 버튼(👍/👎) 클릭
5. 어드민 화면: 지식 등록 (파일 업로드 → 청크 미리보기 → 등록)

### 발표자 노트
> "이게 실제로 동작하는 화면입니다. 답변이 나오는 동시에 어떤 문서를 근거로 했는지도 표시됩니다."
