## [슬라이드 3-1] KnowledgeRAG — 구조와 동작 원리

### 제목
**KnowledgeRAG — 운영 지식을 AI가 답한다**

### 본문 내용

**무엇을 하는가**
- 사내 운영 문서·절차·FAQ를 등록하면 자연어 질문으로 즉시 답변, 근거 문서까지 함께 제시
- 예: 장애 대응("쿠폰 시스템 장애 시 1차 확인 항목은?") · 운영 절차("배송 지연 처리 프로세스는?") · 도메인 용어("DS가 뭐야?") · 반복 문의("PAY_4023 원인은?")

**어떻게 동작하는가 — ① 등록 · ② 검색·답변 · ③ 개선, 3단계가 선순환 구조로 연결**

```
                [① 지식 등록]  PDF·Excel·Confluence·URL·텍스트
                      (등록 전 미리보기 — Human-in-the-loop)
                              │
                              ▼
사용자 질문
    │
    ▼
┌──────────────────────────────────────────────────────────┐
│  Frontend  (React + nginx) — 채팅 UI · 지식 관리 어드민     │
└────────────────────────┬─────────────────────────────────┘
                         │  REST API / SSE 스트리밍
                         ▼
┌──────────────────────────────────────────────────────────┐
│  Backend  (FastAPI)                                       │
│  [② 검색·답변] 용어 매핑 → 하이브리드 검색(벡터×0.7+키워드×0.3)│◀─▶ PostgreSQL
│  → Top-K 선별 → Few-shot 병합 → LLM 답변 생성(SSE 스트리밍)  │   + pgvector
│  [③ 개선] 로그·피드백(👍/👎) 수집 → 지식 가중치 자동 조정     │   (지식·벡터·로그)
│  → 지식 공백 감지 → 어드민 대시보드 가시화                   │◀─▶ 사내 LLM
└────────────────────────┬─────────────────────────────────┘   (DevX Gateway)
                         │
                         ▼
              자연어 답변 + 근거 문서
```

### 발표자 노트
> "오늘 집중적으로 소개드릴 에이전트가 KnowledgeRAG입니다.
> 구조 자체는 프론트-백엔드-DB로 단순하지만, 핵심은 등록·검색·개선이 하나의 선순환 구조로 연결된다는 겁니다.
> 질문이 들어오면 용어 매핑과 하이브리드 검색을 거쳐 답변이 생성되고, 그 피드백이 다시 지식 품질 개선에 쓰입니다.
> 이 구조를 어떻게 만들었는지 지금부터 하나씩 살펴보겠습니다."

## [슬라이드 3-2] 시연

### 제목
**실제 동작 화면**

### 구성
- **[영상 삽입]** 녹화된 시연 영상 재생 (2~3분)

### 시연 시나리오 (영상 녹화 시 참고)
1. 사용자가 운영 이슈 관련 질문 입력
   - 예: "결제 오류 발생 시 1차 확인 항목은 무엇인가요?"
2. 실시간 스트리밍으로 답변 생성되는 모습
3. 답변 하단의 근거 문서 확인
4. 피드백 버튼(👍/👎) 클릭
5. 어드민 화면: 지식 등록 (파일 업로드 → 청크 미리보기 → 등록)

### 발표자 노트
> "이게 실제로 동작하는 화면입니다. 답변이 나오는 동시에 어떤 문서를 근거로 했는지도 표시됩니다."
