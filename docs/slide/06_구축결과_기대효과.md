## [슬라이드 6-1] 구축된 것

### 제목
**Ops-Navigator — 현재 구현된 기능**

### 본문 내용

**채팅 & 에이전트**
- KnowledgeRAG 에이전트: 등록된 지식 기반 자연어 Q&A
- SSE 스트리밍 답변 (실시간 타이핑 효과)
- 대화 히스토리 저장 + 멀티턴 문맥 유지

**지식 등록 (인제스천)**
- 파일 업로드: PDF, Excel(.xlsx/.xlsm), CSV, Markdown, TXT
- Confluence 연동: 단일 페이지 / 하위 페이지 트리 일괄 수집
- URL 크롤링: 일반 웹 페이지 자동 파싱
- 한글 EUC-KR/UTF-8 자동 감지, 암호화 PDF 거부 안내
- 등록 전 청크 미리보기 + 선택 등록 (Human-in-the-loop)

**검색 & 품질**
- 2단계 하이브리드 검색 (벡터 × 0.7 + 키워드 × 0.3)
- 용어집(Glossary) 기반 사내 용어 자동 매핑
- Few-shot Q&A 자동 생성 + 검색 보강
- 신뢰도 기반 답변 필터링 (점수 임계값)
- CrossEncoder 리랭커 (폐쇄망 환경: 모델 탑재 후 활성화)
- 지식 신선도 Decay (오래된 지식 점수 자동 감쇠)

**어드민 & 운영**
- 지식 / 용어집 / Q&A 관리 테이블 (검색, 수정, 다건 삭제)
- 사용자/부서 관리, 역할 기반 권한(RBAC)
- 인제스천 작업 이력 조회
- 지식 갭 감지 — 답 못 찾은 질문 자동 기록 (`no_knowledge` 상태)
- 피드백(👍/👎) → 지식 가중치 자동 반영

**인프라**
- Docker Compose 단일 서버 배포
- PostgreSQL + pgvector (벡터 DB 분리 없이 단일 DB)
- 폐쇄망 배포 지원 (tar.gz 이미지 반입 방식)

### 발표자 노트
> "기능을 나열한 것처럼 보이지만, 각 기능이 실제로 연결되어서 하나의 개선 루프를 만드는 것이 핵심입니다."

## [슬라이드 6-2] 기대효과 — 지금 그리고 데이터가 쌓이면

### 제목
**이 시스템이 팀에 가져올 수 있는 변화**

### 본문 내용

**지금 당장 — 운영 효율화**

```
[Before]               [After]
  질문 → 사람 찾기       질문 → 시스템 즉시 답변
  문서 → 검색 어려움     문서 → 등록 후 자동 검색
  지식 → 개인 의존       지식 → 조직 자산화
  공백 → 모름           공백 → 가시화 + 채움
```

- 반복되는 운영 문의에 대한 즉각적인 1차 답변 제공
- 숙련자 부재 시에도 지식 접근 가능 — 온보딩 비용 절감
- 어떤 주제가 지식 공백인지 데이터로 파악 → 문서화 문화 자연 형성

---

**데이터가 쌓이면 — 열리는 것들**

```
지식 파이프라인이 성숙하면
  ↓
우리 팀만의 허브 데이터
  ├── VOC/VOP 분석
  │     └─ 반복되는 민원·장애 패턴 자동 탐지
  │          "이 오류는 최근 한 달간 12번 물어봤습니다"
  │
  ├── 타 시스템 연계
  │     └─ MCP 도구 확장 → 실시간 로그·DB 조회 결합
  │          단순 Q&A를 넘어 시스템 상태까지 답변
  │
  └── 우리 팀 허브 데이터
        └─ 지식·용어집·Q&A·질의 로그가 한 곳에 축적
             운영 노하우가 사람이 아닌 시스템에 남는다
```

| 가능성 | 내용 |
|--------|------|
| **VOC/VOP 운영 효율화** | 쌓인 질의 로그에서 반복 패턴 탐지 → "이 주제는 매주 문의가 옵니다" 선제 대응 |
| **실시간 시스템 연계** | MCP 도구로 모니터링 API·로그 시스템 연결 → 지식 + 실시간 데이터 결합 답변 |
| **우리 팀만의 도메인 허브** | 지식·용어집·Q&A·질의 로그가 축적될수록 → 운영 노하우가 사람이 아닌 시스템에 남는 구조 |

### 발표자 노트
> "지금 당장의 효과는 반복 문의 처리와 지식 체계화입니다.
> 하지만 더 중요한 건 데이터가 쌓이면서 열리는 가능성입니다.
> 질의 로그가 축적되면 VOC처럼 '어떤 문제가 자주 생기는지' 패턴을 잡을 수 있고,
> MCP 도구로 실시간 시스템 데이터를 연결하면 단순 Q&A를 넘어설 수 있습니다.
> 결국 이 파이프라인이 우리 팀 운영 도메인 지식의 허브가 되는 게 목표입니다."
